● Correção de privacidade

Cada pessoa com sua própria conta

Seu amigo viu seus dados porque o app guardava tudo numa única gaveta compartilhada. Agora cada usuário tem a sua — e você e o Lucas continuam juntos.

Por que acontecia

No código, todo mundo lia e gravava a mesma linha do banco: id = "casal". Não importava quem logasse — caía sempre nos seus dados. Pior: se o seu amigo salvasse algo, sobrescrevia os seus.

A correção troca esse id fixo por um id por conta: cada pessoa nova ganha uma gaveta isolada; você e o Lucas ficam ligados à gaveta casal.

1Rode o SQL no Supabase

É a parte que realmente protege os dados (o banco passa a proibir um usuário de ver a conta do outro). Abra o Supabase → SQL Editor → cole o conteúdo do arquivo e rode.

📄 supabase_isolar_contas.sql
No PASSO 4 do arquivo, troque os textos pelos UID reais da Thayse e do Lucas — você pega em Authentication → Users, coluna UID. É isso que mantém vocês dois na mesma conta.

2Depois, suba o código

Com o SQL já rodado, descompacte o pacote na raiz do projeto e:

git add .
git commit -m "fix: isolar contas por usuario (privacidade)"
git push

3Confira

✓Você e o Lucas entram e veem os dados de sempre (a conta "casal").
✓Seu amigo entra e vê uma conta vazia, só dele.
✓Ninguém mais enxerga ou altera a conta de ninguém.
Importante — ordem: rode o SQL antes do git push. Assim você e o Lucas nunca chegam a ver a tela vazia. Se algum de vocês logar antes do PASSO 4, é só rodá-lo depois — ele corrige o vínculo automaticamente.
